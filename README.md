# Práctica de Trabajo Colaborativo con Git y GitHub

## Descripción del Proyecto - (RAMA DEV)

Este proyecto (creado por dev) consiste en el desarrollo de una página web utilizando HTML y CSS con el propósito de aplicar herramientas y metodologías de trabajo colaborativo mediante Git y GitHub.

Durante el desarrollo se implementaron ramas de trabajo, control de versiones, integración de cambios mediante Pull Requests y resolución de conflictos, permitiendo que varios integrantes trabajaran simultáneamente sobre el mismo proyecto.

## Objetivo del Proyecto

Aplicar los conceptos de control de versiones y trabajo colaborativo utilizando Git y GitHub, mediante la creación de un proyecto web sencillo en el que los integrantes del equipo desempeñen distintos roles y sigan un flujo de trabajo estructurado.

## Tecnologías Utilizadas

## Estructura de Ramas

El proyecto utiliza la siguiente estructura de ramas:

* **main**: Rama principal que contiene la versión estable del proyecto.
* **dev**: Rama destinada al desarrollo y documentación.
* **diseno**: Rama utilizada para realizar cambios relacionados con la apariencia visual del sitio.

## Flujo de Trabajo Utilizado

1. Se creó el repositorio en GitHub.
2. Se definieron los roles de cada integrante.
3. Se creó la rama principal `main`.
4. Se crearon las ramas de desarrollo `dev` y `diseno`.
5. Cada integrante trabajó en su rama correspondiente realizando commits descriptivos.
6. Los cambios fueron enviados al repositorio remoto mediante Git.
7. Se generaron Pull Requests para solicitar la integración de cambios.
8. Los Pull Requests fueron revisados por otro integrante antes de ser aceptados.
9. Se realizó la integración de cambios a la rama principal.
10. Se resolvieron conflictos de fusión para comprender el proceso de integración colaborativa.
