# Práctica de Trabajo Colaborativo con Git y GitHub DUNE

## Descripción del Proyecto

Este proyecto consiste en el desarrollo de una página web inspirada en el universo de Dune utilizando HTML, CSS y JavaScript. El sitio presenta información relacionada con la saga y fue desarrollado como parte de una práctica de trabajo colaborativo utilizando Git y GitHub.

Durante el desarrollo se emplearon ramas de trabajo, control de versiones, Pull Requests y resolución de conflictos para simular un entorno de desarrollo colaborativo.

## Objetivo del Proyecto

Aplicar los conceptos de control de versiones y trabajo colaborativo mediante el uso de Git y GitHub, desarrollando una página web temática sobre Dune en la que los integrantes del equipo desempeñan distintos roles y siguen un flujo de trabajo organizado.

## Flujo de Trabajo Utilizado

Para el desarrollo del proyecto se utilizó una estrategia basada en ramas. Se creó una rama principal llamada **main**, encargada de almacenar la versión estable del proyecto. Además, se crearon dos ramas de desarrollo: **dev**, destinada a cambios relacionados con el desarrollo y la documentación, y **diseno**, utilizada para las modificaciones visuales de la página.

Cada integrante trabajó en su respectiva rama realizando commits descriptivos para registrar los cambios efectuados. Posteriormente, los cambios fueron enviados al repositorio remoto mediante Git y se utilizaron Pull Requests para solicitar la integración de las modificaciones a la rama principal.

Antes de realizar las fusiones, los cambios fueron revisados por otro integrante del equipo. Asimismo, se generó y resolvió un conflicto de fusión con el propósito de comprender el proceso de integración de cambios cuando distintas ramas modifican una misma sección del proyecto.
