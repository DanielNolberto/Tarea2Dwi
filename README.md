Práctica de Trabajo Colaborativo con Git y GitHub DUNE
Descripción del Proyecto

Este proyecto consiste en el desarrollo de una página web con temática de la película y universo de Dune utilizando HTML, CSS y JavaScript, con el propósito de aplicar herramientas y metodologías de trabajo colaborativo mediante Git y GitHub.

Durante el desarrollo se implementaron ramas de trabajo, control de versiones, integración de cambios mediante Pull Requests y resolución de conflictos, permitiendo que varios integrantes trabajaran simultáneamente sobre el mismo proyecto.

Objetivo del Proyecto

Aplicar los conceptos de control de versiones y trabajo colaborativo utilizando Git y GitHub mediante la creación de un proyecto web temático sobre Dune, donde los integrantes del equipo desempeñan distintos roles y siguen un flujo de trabajo estructurado.

Estructura de Ramas

El proyecto utiliza la siguiente estructura de ramas:

main: Rama principal que contiene la versión estable del proyecto.
dev: Rama destinada al desarrollo y documentación.
diseno: Rama utilizada para realizar cambios relacionados con la apariencia visual del sitio.
Flujo de Trabajo Utilizado
Se creó el repositorio en GitHub.
Se definieron los roles de cada integrante.
Se creó la rama principal main.
Se crearon las ramas de desarrollo dev y diseno.
Cada integrante trabajó en su rama correspondiente realizando commits descriptivos.
Los cambios fueron enviados al repositorio remoto mediante Git.
Se generaron Pull Requests para solicitar la integración de cambios.
Los Pull Requests fueron revisados por otro integrante antes de ser aceptados.
Se realizó la integración de cambios a la rama principal.
Se resolvieron conflictos de fusión para comprender el proceso de integración colaborativa.
